Audit scope, in priority order:

1. ENCODING INTEGRITY
   Scan for mojibake: UTF-8 text misread as Windows-1252 at some point and
   re-saved, corrupting em dashes / curly quotes / box-drawing characters
   into garbage byte sequences. This breaks PowerShell parsing and looks bad
   in shipped docs. Report file + line, then give me a fix script.
   Also scan for BOM-prefixed files (UTF-8 BOM or otherwise). This is a
   Next.js/TypeScript repo, so a leading BOM breaking JSON.parse or the
   TS/JS parser is at least as likely a failure mode as Windows-1252
   round-trip corruption. Report file + detected BOM type, then give me a
   fix script to strip it.

2. SECRETS AND CREDENTIALS

   not sourced from an environment variable or secret store. Tell me the
   file, line, and pattern type only. Do not show me the actual secret value

5. COMPLEXITY HOTSPOTS
   Long methods, deep nesting, high branching. Exclude generated files.
   Rank by risk, not just length: a 300-line method on a payment/fiscal/auth
   path matters more than a 300-line method in a report formatter.

6. SECURITY REVIEW

   unsafe file handling, sensitive data in logs.

8. PRODUCTION READINESS
